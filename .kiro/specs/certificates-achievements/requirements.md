## 🎯 الهدف

توفير نظام شامل للشهادات الرقمية والإنجازات لتحفيز المستخدمين وزيادة قيمة الدورات التدريبية، مع إمكانية التحقق من صحة الشهادات ومشاركتها على المنصات المهنية.

---

## 👥 المستخدمون المستهدفون

1. **المتدربون** - للحصول على شهادات معترف بها
2. **الشركات** - للتحقق من صحة الشهادات
3. **مقدمو الدورات** - لإصدار شهادات احترافية
4. **مسؤولو التوظيف** - للتحقق من مؤهلات المرشحين

## 📝 User Stories

### 1. إصدار الشهادة التلقائي

**كمتدرب:**
- أريد الحصول على شهادة تلقائياً عند إكمال الدورة
- أريد أن تحتوي الشهادة على اسمي واسم الدورة والتاريخ
- أريد شهادة بتصميم احترافي وجذاب
- أريد تحميل الشهادة بصيغة PDF عالية الجودة
- أريد استلام إشعار فوري عند إصدار الشهادة

**معايير القبول:**
- [x] الشهادة تُصدر تلقائياً عند إكمال 100% من الدورة
- [x] الشهادة تحتوي على: الاسم، الدورة، التاريخ، رقم فريد
- [x] تصميم احترافي مع شعار Careerak
- [x] تحميل PDF بجودة عالية (300 DPI)
- [x] إشعار فوري + بريد إلكتروني

### 2. QR Code للتحقق

**كمتدرب:**
- أريد QR Code على الشهادة للتحقق من صحتها
- أريد أن يكون الـ QR Code فريد لكل شهادة
- أريد أن يؤدي مسح الـ QR Code إلى صفحة تحقق رسمية

**كشركة:**
- أريد مسح QR Code للتحقق من صحة الشهادة
- أريد رؤية تفاصيل الشهادة (المتدرب، الدورة، التاريخ)
- أريد التأكد من أن الشهادة لم يتم إلغاؤها أو تزويرها

**معايير القبول:**
- [x] كل شهادة لها QR Code فريد
- [x] QR Code يحتوي على رابط التحقق
- [x] صفحة التحقق تعرض جميع تفاصيل الشهادة
- [x] التحقق يعمل بدون تسجيل دخول
- [x] عرض حالة الشهادة (صالحة، ملغاة، منتهية)

## 🔧 المتطلبات التقنية

### Backend

1. **Certificate API**
   - `POST /certificates/generate` - إصدار شهادة
   - `GET /certificates` - جلب شهادات المستخدم
   - `GET /certificates/:id` - جلب شهادة واحدة
   - `GET /certificates/verify/:code` - التحقق من شهادة
   - `PUT /certificates/:id/revoke` - إلغاء شهادة
   - `POST /certificates/:id/reissue` - إعادة إصدار

2. **Badge API**
   - `GET /badges` - جلب جميع الـ badges
   - `GET /badges/user/:userId` - badges المستخدم
   - `POST /badges/award` - منح badge
   - `GET /badges/progress` - تقدم المستخدم

3. **Verification API**
   - `GET /verify/:certificateCode` - صفحة التحقق العامة
   - `POST /verify/bulk` - التحقق من عدة شهادات

4. **LinkedIn Integration**
   - `POST /integrations/linkedin/share` - مشاركة على LinkedIn
   - OAuth 2.0 authentication

5. **Database Models**
   - Certificate (الشهادة)
   - Badge (الإنجاز)
   - UserBadge (badges المستخدم)
   - CertificateTemplate (قوالب الشهادات)

### Frontend

1. **Certificate Display**
   - عرض الشهادة بتصميم احترافي
   - تحميل PDF
   - مشاركة على LinkedIn

2. **Certificates Gallery**
   - معرض الشهادات
   - فلترة وترتيب
   - إخفاء/إظهار

3. **Badges Display**
   - عرض الـ badges
   - تقدم الإنجازات
   - شرح كيفية الحصول على كل badge

4. **Verification Page**
   - صفحة التحقق العامة
   - مسح QR Code
   - عرض النتائج
